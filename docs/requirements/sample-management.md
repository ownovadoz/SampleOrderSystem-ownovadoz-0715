# 시료 관리 모듈 요구사항

이 문서는 시료 관리 모듈 개발을 담당하는 에이전트를 위한 문서입니다. 시스템 공통 배경, 도메인 모델(시료/
주문 필드, 주문 상태, 재고 선점 규칙), 시간/영속성 정책, 아키텍처·개발 공통 규칙은 `PRD.md`를 먼저 읽고
따릅니다. 이 문서에는 시료 관리 모듈에만 해당하는 요구사항만 적습니다.

각 항목 끝의 "출처"는 사람이 원문을 다시 확인해 검증할 수 있도록, 어떤 문서의 어느 부분을 근거로 정리했는지
표시한 것입니다.

## 목적

> **핵심 요약**: 시료는 시스템의 최소 단위이며, 등록된 시료만 주문할 수 있다.

시료(Sample)는 이 시스템의 가장 기본이 되는 단위이며, 각 시료는 고유한 이름과 속성을 가진다. 시스템에 등록된
시료만 주문 가능하다.

> 출처: `docs/requirements.pdf` p.12 (시료 관리 — "시료(Sample)는 이 시스템의 가장 기본이 되는 단위",
> "시스템에 등록된 시료만 주문 가능")

## 메뉴 구성

| 메뉴 | 설명 |
|---|---|
| 시료 등록 | 새로운 시료를 시스템에 추가 |
| 시료 조회 | 등록된 모든 시료 목록을 확인, 현재 재고 수량도 함께 표시 |
| 시료 검색 | 이름 등 속성으로 특정 시료를 검색 |

> 출처: `docs/requirements.pdf` p.10 (메인 메뉴 — "시료 관리: 새로운 시료 등록, 목록 조회, 이름 검색 기능"),
> p.12 (시료 관리 — 메뉴별 상세)

## 상세 규칙

### 시료 등록

> **핵심 요약**: 시료 ID·이름·평균 생산시간·수율을 입력받아 등록. ID 중복 금지, 수율은 0 초과 1 이하.

입력 값:
- 시료 ID
- 이름
- 평균 생산시간
- 수율 (0 초과 1 이하 범위로 검증. 0 또는 음수, 1 초과 값은 등록을 거부한다)

시료 ID는 시스템 내에서 유일해야 한다. 이미 존재하는 ID로 등록을 시도하면 거부하고 사유를 안내한다.

> 출처: `docs/requirements.pdf` p.12 (시료 관리 — "속성 값 : 시료 ID, 이름, 평균 생산시간, 수율", 수율 정의
> "(정상적인 시료 / 총 생산 시료)"). ID 유일성/수율 범위 검증은 명세에 명시되어 있지 않은 부분으로,
> 데이터 무결성을 위해 이 문서에서 추가로 정한 규칙임 — 구현 시 재검토 가능.

### 시료 조회

> **핵심 요약**: 등록된 시료 전체 + 재고. 이때 재고는 선점 여부와 무관한 원재고.

- 등록된 모든 시료를 목록으로 표시한다.
- 각 시료의 현재 재고 수량을 함께 표시한다. 이때 표시하는 재고는 **선점 여부와 무관한 원재고**다
  (다른 주문이 선점했는지는 이 화면에 영향을 주지 않는다 — 상세 근거는 `PRD.md` 2.4절 참고).

> 출처: `docs/requirements.pdf` p.12 (시료 관리 — "시료 조회: 등록된 모든 시료 목록을 확인, 현재 재고
> 수량도 함께 표시"), p.13 (예시 UI 화면 — ID/시료명/평균 생산시간/수율/현재 재고 표) / `docs/memo.txt`
> 18번째 줄 ("다만, 선점돼있는 재고더라도 재고 현황에서는 재고로 보여야 한다")

### 시료 검색

> **핵심 요약**: 이름 등 속성으로 검색.

- 이름 등 속성으로 특정 시료를 검색해 목록 형태로 결과를 표시한다. 검색 방식(부분 일치/대소문자 무시 등)은
  합리적으로 자율 결정한다.

> 출처: `docs/requirements.pdf` p.12 (시료 관리 — "시료 검색: 이름 등 속성으로 특정 시료를 검색")

## 다른 모듈과의 인터페이스 계약

- 주문 관리 모듈은 주문 접수 시 시료 ID의 존재 여부를 확인해야 하므로, 시료 관리 모듈은 "시료 ID로 시료
  존재 여부 및 속성(이름, 평균 생산시간, 수율)을 조회"하는 기능을 다른 모듈이 사용할 수 있는 형태로
  제공해야 한다.
- 모니터링 모듈은 시료별 재고 현황을 다시 표시해야 하므로, "등록된 시료 목록 + 재고 수량 조회" 기능 역시
  다른 모듈에서 재사용 가능해야 한다.
