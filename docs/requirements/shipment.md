# 출고 처리 모듈 요구사항

이 문서는 출고 처리 모듈 개발을 담당하는 에이전트를 위한 문서입니다. 시스템 공통 배경, 도메인 모델(시료/
주문 필드, 주문 상태, 재고 선점 규칙), 시간/영속성 정책, 아키텍처·개발 공통 규칙은 `PRD.md`를 먼저 읽고
따릅니다. 이 문서에는 출고 처리 모듈에만 해당하는 요구사항만 적습니다.

각 항목 끝의 "출처"는 사람이 원문을 다시 확인해 검증할 수 있도록, 어떤 문서의 어느 부분을 근거로 정리했는지
표시한 것입니다.

## 목적

> **핵심 요약**: CONFIRMED 상태 주문을 사람이 직접 출고 처리하는 기능.

재고가 충분해져 `CONFIRMED` 상태가 된 주문에 대해, 사람이 직접 출고를 처리한다.

> 출처: `docs/requirements.pdf` p.22 (출고 처리 — "재고가 충분해진 CONFIRMED 주문에 대하여 출고를 처리")
> / `docs/memo.txt` 16번째 줄 ("출고는 사람이 직접하므로 생산이 완료됐다고 released로 변경하면 안된다")

## 메뉴 구성

| 메뉴 | 설명 |
|---|---|
| 출고 가능 주문 목록 | `CONFIRMED` 상태의 주문 목록 표시 |
| 출고 처리 | 특정 주문을 선택해 출고 실행 |

> 출처: `docs/requirements.pdf` p.10 (메인 메뉴 — "출고 처리: CONFIRMED 상태 주문에 대해 출고 실행"), p.23
> (예시 UI — 출고 가능 주문 표)

## 상세 규칙

### 출고 가능 주문 목록

> **핵심 요약**: CONFIRMED 상태 주문만 표시.

`CONFIRMED` 상태의 주문만 목록으로 표시한다.

> 출처: `docs/requirements.pdf` p.23 (예시 UI — "출고 가능 주문 (CONFIRMED)" 표)

### 출고 처리

> **핵심 요약**: CONFIRMED → RELEASE 전환 + 재고 실제 차감. 생산 완료만으로는 절대 자동 전환되지 않는다.

- 특정 `CONFIRMED` 주문을 선택해 출고를 실행한다.
- 처리 시 주문 상태를 `CONFIRMED` → `RELEASE`로 전환한다.
- 출고는 재고를 물리적으로 소비하는 시점이다: 해당 시료의 재고에서 주문 수량만큼 **실제로 차감**한다
  (선점되어 있던 재고가 실제로 빠져나가는 시점 — `PRD.md` 2.4절 참고).
- 생산 완료만으로는 절대 `RELEASE`로 전환되지 않는다 (사람이 이 메뉴에서 직접 실행해야만 전환된다).

> 출처: `docs/requirements.pdf` p.22 (출고 처리 — "특정 주문에 대해 출고를 실행", "주문 상태가 RELEASE로
> 전환"), p.8 (주문 상태 흐름 — "RELEASE: 출고 완료") / `docs/memo.txt` 16~17번째 줄 ("출고는 사람이
> 직접하므로 생산이 완료됐다고 released로 변경하면 안된다", "어쨌든 재고는 이미 선점되어있는 상태이기
> 때문에 재고 계산 등에는 문제 없음" — 즉 재고 차감은 출고 시점에 실제로 일어남을 뒷받침)

## 예시 UI 참고

`docs/requirements.pdf` p.23 (출고 처리)을 참고한다. 다만 화면 문구/표 형식 등 구체적인 표현은 자유롭게
결정해도 된다.

## 다른 모듈과의 인터페이스 계약

- 주문 관리 모듈이 정의한 주문 데이터/상태를 조회하고 갱신(`CONFIRMED` → `RELEASE`)할 수 있어야 한다.
- 시료 관리 모듈이 관리하는 재고 데이터를 차감할 수 있어야 한다.
