# 모니터링 모듈 요구사항

이 문서는 모니터링 모듈 개발을 담당하는 에이전트를 위한 문서입니다. 시스템 공통 배경, 도메인 모델(시료/
주문 필드, 주문 상태, 재고 선점 규칙), 시간/영속성 정책, 아키텍처·개발 공통 규칙은 `PRD.md`를 먼저 읽고
따릅니다. 이 문서에는 모니터링 모듈에만 해당하는 요구사항만 적습니다.

각 항목 끝의 "출처"는 사람이 원문을 다시 확인해 검증할 수 있도록, 어떤 문서의 어느 부분을 근거로 정리했는지
표시한 것입니다.

## 목적

> **핵심 요약**: 담당자가 주문/재고 현황을 한눈에 파악하도록 돕는 조회 전용 기능.

주문에 따른 시료 재고에 대한 정보를 모니터링하는 기능. 담당자가 현재 시스템 상태를 한눈에 파악할 수 있도록
구성한다.

> 출처: `docs/requirements.pdf` p.18 (모니터링 — "주문에 따른 시료 재고에 대한 정보 모니터링 기능",
> "담당자가 현재 시스템의 상태를 한눈에 파악할 수 있도록 구성")

## 메뉴 구성

| 메뉴 | 설명 |
|---|---|
| 주문량 확인 | 상태별 주문 수 확인 |
| 재고량 확인 | 시료별 재고 현황 확인 |

> 출처: `docs/requirements.pdf` p.10 (메인 메뉴 — "모니터링: 상태별 주문 수 및 시료별 재고 현황 확인"),
> p.18 (메뉴별 상세)

## 상세 규칙

### 주문량 확인

> **핵심 요약**: 상태별(RESERVED/CONFIRMED/PRODUCING/RELEASE) 건수 집계, REJECTED는 항상 제외.

- 상태별(`RESERVED` / `CONFIRMED` / `PRODUCING` / `RELEASE`) 주문 건수를 집계해 표시한다.
- `REJECTED`는 유효한 주문이 아니므로 이 집계에서 항상 제외한다.

> 출처: `docs/requirements.pdf` p.18 (모니터링 — "현재 상태별(RESERVED/CONFIRMED/PRODUCING/RELEASE) 목록을
> 확인", "REJECTED 는 유효한 주문이 아니므로 무시"), p.19 (예시 UI — 상태별 주문 현황 표)

### 재고량 확인

> **핵심 요약**: 시료별 원재고 표시 + 여유/부족/고갈 상태 라벨.

- 각 시료별 현재 재고 수량을 표시한다. 이때 표시하는 재고는 **선점 여부와 무관한 원재고**다 (`PRD.md`
  2.4절 참고 — 신규 주문 승인 판단에 쓰이는 "가용 재고"와는 다른 값이다. 이 화면은 항상 원재고를 그대로
  보여준다).
- 주문 대비 재고 수량에 따라 상태를 함께 표기한다:
  - **여유**: 주문 대비 재고가 충분한 상태
  - **부족**: 주문 대비 재고 수량이 부족한 상태
  - **고갈**: 재고 수량이 0인 상태

  "여유/부족"을 가르는 구체적인 수치 기준은 명세에 명시되어 있지 않으므로, 합리적인 기준(예: 미출고
  주문 총 수요 대비 원재고 비율 등)을 정하고 **그 기준을 코드 주석이 아닌 이 모듈의 문서/커밋 메시지에
  명시적으로 남긴다.**

> 출처: `docs/requirements.pdf` p.18 (모니터링 — "각 시료별 현재 재고 수량을 확인", "여유 : 주문대비 재고
> 충분 상태 / 부족 : 주문대비 재고 수량 부족 상태 / 고갈 : 수량이 0인 상태"), p.19 (예시 UI — 재고 현황
> 표, 잔여율 바) / `docs/memo.txt` 18~19번째 줄 ("선점돼있는 재고더라도 재고 현황에서는 재고로 보여야 함",
> "재고 파악때는 재고 그대로"). 여유/부족을 가르는 정확한 임계값은 두 문서 모두 정의하지 않은 부분임.

## 예시 UI 참고

`docs/requirements.pdf` p.19 (모니터링)을 참고한다. 다만 화면 문구/표 형식 등 구체적인 표현은 자유롭게
결정해도 된다.

## 다른 모듈과의 인터페이스 계약

- 주문 관리 모듈로부터 상태별 주문 목록/건수 조회 기능을 사용한다.
- 시료 관리 모듈로부터 시료별 재고(원재고) 조회 기능을 사용한다.
- 이 모듈은 조회 전용(read-only)이며, 어떤 도메인 상태도 직접 변경하지 않는다.
